## 1. Purpose
Define administrative controls for secure and compliant EDMS configuration and maintenance.

## 2. Scope
Applies to system administrators, IT support, validation, and QA oversight personnel.

## 3. Responsibilities
- **System Administrator:** execute approved configuration, user management, and maintenance tasks.
- **QA:** review/approve compliance-impacting changes.
- **IT Security:** enforce identity, access, and security controls.

## 4. Procedure
### 4.1 User and Role Management
1. Provision/deprovision accounts via approved access request.
2. Assign least-privilege roles.
3. Conduct periodic access review (at least quarterly).

### 4.2 Workflow and Template Configuration
1. Configure workflows/templates only under approved change control.
2. Verify segregation of duties and authority checks.
3. Retest impacted controls per risk assessment.

### 4.3 Backup, Restore, and Archival
1. Perform scheduled backups and monitor completion.
2. Execute periodic restore challenge tests.
3. Verify archival readability and retrieval.

### 4.4 Audit Trail Monitoring
1. Ensure audit logging is continuously enabled.
2. Review privileged action logs on a defined schedule.
3. Escalate anomalies per incident management process.

### 4.5 Incident and Change Management
1. Log incidents/defects with impact categorization.
2. Route changes through formal approval workflow.
3. Complete revalidation impact assessment before release.

## 5. Records
- Access requests and approvals
- Role change logs
- Change control records
- Backup/restore evidence
- Audit review logs
